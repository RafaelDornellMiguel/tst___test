Guia Prático: Abertura de Tickets ETL Migração

Suporte para o time de CEOps

Introdução

Este guia foi desenvolvido para auxiliar o time de CEOps Suporte no preenchimento correto dos formulários de tickets para ETL Migração. Siga as instruções abaixo para garantir que todas as informações necessárias sejam fornecidas corretamente, agilizando o processo de atendimento.

Perguntas Frequentes

O que é ETL Migração?

ETL (Extract, Transform, Load) Migração refere-se ao processo de transferência de dados de um sistema para outro, envolvendo extração de dados do sistema de origem, transformação dos dados para o formato adequado e carregamento no sistema de destino.

Por que precisamos de informações precisas?

Tickets completos e precisos permitem que nossa equipe de ETL Migração atenda às solicitações mais rapidamente, evitando atrasos e retrabalho.

Fluxo de Trabalho

  1. CEOps Suporte recebe a solicitação do cliente
  2. Preenche o formulário com todas as informações necessárias
  3. Time de ETL Migração recebe o ticket
  4. Análise e execução da migração
  5. Feedback e conclusão
Atenção!

Campos marcados com asterisco (*) são obrigatórios e devem ser preenchidos com atenção especial.

Preenchimento do Formulário: Campo a Campo

Origem ETL *

Selecione a origem dos dados que serão migrados. Este campo é crucial para que a equipe de ETL entenda de onde virão os dados.

Como preencher:

Selecione uma das opções disponíveis no menu suspenso. Caso não saiba a origem, consulte o cliente antes de enviar o ticket.

ID do sistema *

Informe o identificador único do sistema de origem. Este código é essencial para localizar corretamente os dados a serem migrados.

Atenção:

O ID do sistema deve ser digitado exatamente como aparece no sistema de origem, respeitando letras maiúsculas, minúsculas e caracteres especiais.

Responsável pela abertura do ticket *

Insira seu nome completo para facilitar a comunicação interna. Você será o ponto de contato para qualquer dúvida relacionada a este ticket.

E-mail do responsável da clínica *

Informe o e-mail do cliente que solicitou a migração. Este é o e-mail que será usado para comunicações sobre o andamento do processo.

Dica:

Confirme que o e-mail está correto antes de submeter o formulário. Um e-mail incorreto pode causar atrasos significativos.

Nome do responsável da clínica *

Digite o nome completo do responsável pela clínica que está solicitando a migração.

Telefone responsável da clínica *

Forneça o número de telefone do responsável, incluindo DDD.

Formato recomendado:

XXXXXXXXXXX

Como é o nome do atual software do cliente *

Selecione o software que o cliente utiliza atualmente e do qual os dados serão exportados.

Importante:

A correta identificação do software atual é crucial para determinar o método de extração de dados.

Tipo de importação *

Selecione o tipo de importação que será realizada. Cada tipo tem requisitos e procedimentos específicos.

Tipos comuns:
  • Migração completa
  • Migração parcial
  • Atualização de dados
  • Consolidação de bases

Motivo da Importação (campo de observação para detalhar) *

Explique detalhadamente o motivo da solicitação de migração. Inclua todas as informações relevantes que possam ajudar a equipe de ETL.

Exemplo de preenchimento:

"Cliente solicitou migração completa dos dados do sistema XYZ para nosso sistema. Eles necessitam que todos os dados de pacientes e histórico de consultas dos últimos 5 anos sejam migrados com urgência, pois irão descontinuar o uso do sistema atual em 15 dias."

Acesso liberado do cliente? *

Indique se o cliente já liberou acesso ao sistema de origem para extração dos dados.

Atenção:

Se o acesso não estiver liberado, o processo de migração não poderá ser iniciado. É essencial obter essa liberação antes de prosseguir.

Drive com os arquivos de importação *

Forneça o link para o Google Drive ou outro repositório onde estão armazenados os arquivos necessários para a migração.

Dica:

Certifique-se de que o link tenha permissões de acesso corretas e que todos os arquivos necessários estejam incluídos.

Quais dados vai importar: *

Selecione todos os tipos de dados que serão migrados. É possível selecionar múltiplas opções.

Atenção:

Selecionar corretamente os tipos de dados é crucial para que a migração seja bem-sucedida. Caso não tenha certeza, consulte o cliente para confirmar.

Opções disponíveis:

  • Dados cadastrais
  • Ficha clínica
  • Financeiro do paciente
  • Agendamentos
  • Orçamentos em aberto
  • Cadastro de profissionais
  • Arquivos
Concluindo o formulário:

Após preencher todos os campos obrigatórios, clique no botão "Enviar" para submeter o ticket para a equipe de ETL Migração.

Perguntas Frequentes

O que fazer quando não sei o nome do software atual do cliente?

Entre em contato com o cliente e peça essa informação antes de submeter o ticket. O nome correto do software é crucial para o processo de migração.

Por que preciso incluir o ID do sistema?

O ID do sistema é um identificador único que ajuda a equipe de ETL a localizar exatamente o sistema de origem. Sem essa informação, pode haver atrasos significativos no processo de migração.

O cliente não liberou acesso ao sistema. Posso enviar o ticket mesmo assim?

Sim, você pode enviar o ticket, mas é importante selecionar "Não" no campo "Acesso liberado do cliente?" Isso sinalizará para a equipe de ETL que ainda será necessário obter o acesso antes de iniciar a migração.

Qual a diferença entre migração completa e parcial?

Uma migração completa envolve transferir todos os dados do sistema de origem para o novo sistema. Uma migração parcial envolve apenas conjuntos específicos de dados, como apenas cadastros de pacientes ou apenas dados financeiros.

O cliente não tem todos os arquivos necessários. O que devo fazer?

Oriente o cliente sobre quais arquivos são necessários e como obtê-los. Somente após reunir todos os arquivos necessários, o ticket deve ser submetido.

Quanto tempo leva para concluir uma migração?

O tempo varia dependendo da complexidade e volume dos dados, bem como do tipo de migração. Em média, migrações completas podem levar de 1 a 3 semanas, enquanto migrações parciais podem ser mais rápidas.

O cliente mudou de ideia sobre quais dados migrar. Como proceder?

Se o ticket já foi enviado, entre em contato com a equipe de ETL diretamente para informar sobre as mudanças. Se o ticket ainda não foi enviado, atualize as informações antes de submetê-lo.
